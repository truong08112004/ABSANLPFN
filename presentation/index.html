Aspect-Based Sentiment Analysis with Deep Learning: a Multitask Approach

Aspect + sentiment · CNN+BiLSTM vs BERT

Following the paper

Model A implements the MABSA flow: GloVe/POS → CNN(3) → BiLSTM → pooling → multitask heads.

Trang Uyen Tran et al. — integrated CNN–BiLSTM for multitask ABSA

Problem

ABSA: predict what the review is about and how the author feels — from one English review.

  • Input: reviewText (transport-domain reviews)
  • Task 1 — Aspect: multi-class (e.g. Convenience, Staff quality, Facilities)
  • Task 2 — Sentiment: Positive, Neutral, or Negative
  • Multitask: one shared encoder, two heads — context helps both tasks

Multitask learning

Pipeline

  • Same CSV & split for both models
  • Preprocess → token IDs (+ POS for LSTM)
  • Model A: CNN + BiLSTM
  • Model B: fine-tuned BERT
  • Both → two softmax heads + combined loss

Preprocessing

Model A — CNN + BiLSTM

  • 100-d embeddings + one-hot POS
  • 3× Conv1d — local n-gram features
  • BiLSTM — long-range context
  • Avg pool → shared vector → two heads
100 ep · bs 128 · lr 1e-3

Model B — BERT

  • bert-base-uncased — 12 layers, 768-d
  • CLS summarizes the review
  • Same multitask heads as Model A
  • Fine-tune pretrained English → domain reviews
5 ep · bs 64 · lr 2e-5 · max_len 128

Takeaways

  • BERT wins on aspect (+10.6% acc)
  • CNN + BiLSTM lighter, no pretrain
